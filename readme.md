# Description

Built a responsive website, including accessibility elements. This website's index page consists of a hero section, service grids, customer stories. You can find that the semantic HTML elements `<header>`, `<main>`, `<section>`, `<article>`, `<footer>`, etc. and ARIA attributes are used in a well-defined manner. This website is fully responsive and has an apt color contrast to ensure compatibility with screen readers and assistive technologies. Flex, grids, and media queries guarantee seamless performance across desktop, tablet, and mobile devices. Also, you can see that the About Us and Contact form are written in separate html pages.

## How to view/test the website

1. Clone or download the project repository.
2. Open index.html in any browser Chrome, Firefox, Safari, or Edge.
3. Use the dev tools to check the Dimensions responsiveness, accessibility etc.,
4. You can check the contact form for input to see accessibility criteria added.

## Accessibility challenges

Identified issues like poor color contrast and missing ARIA labels. Applied semantic HTML, ARIA attributes, and live regions to ensure screen reader compatibility.

## Responsive & inclusive design

Designed with flexible grids, media queries, and scalable typography. Tested on multiple devices and screen sizes to ensure consistent experience for all users.

## Tools & resources

Used browser DevTools, checked across browsers, used Figma for changing image resolution. Followed all the relevant guidelines.
